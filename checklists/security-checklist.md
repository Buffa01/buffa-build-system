# Security Checklist

- Auth requerida donde corresponde.
- Permisos validados en backend.
- Inputs validados y sanitizados.
- IDs de URL/body validados.
- Uploads validados si aplica.
- Tenant isolation revisada si aplica.
- Secrets fuera del repo.
- Logs sin datos sensibles.
- Rate limits considerados.
- Errores no exponen datos innecesarios.
- Inputs validados.
- Operaciones sensibles auditadas.
- Multi-tenancy revisada.
- Dependencias criticas revisadas.
