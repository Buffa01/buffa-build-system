# Pull Request Checklist

- Cambio alineado a una task.
- Cambios acotados.
- Specs actualizadas si cambio comportamiento.
- Tests agregados o actualizados.
- Lint/typecheck ejecutados.
- Permisos revisados.
- Seguridad revisada.
- No hay secrets.
- No hay logs sensibles.
- No hay archivos gigantes.
- No hay funciones gigantes.
- No hay dependencias innecesarias.
- Build/lint/test ejecutados.
- Riesgos documentados.
