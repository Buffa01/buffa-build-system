# Onboarding

Guia para una persona o agente que entra al proyecto.

## Leer primero

1. README.
2. Product vision.
3. Global rules.
4. Architecture.
5. Roadmap.
6. Modulos activos.
7. Status actual.
8. Source project analysis si se esta mejorando la base.

## Entender

- Que problema resuelve.
- Quienes usan el producto.
- Que modulos son core.
- Que reglas no se pueden romper.
- Como se prueba.
