# Project Health Check

Checklist para revisar salud del proyecto.

## Producto

- Vision clara.
- Roadmap actualizado.
- Verticales con valor.

## Tecnologia

- Build estable.
- Tests criticos.
- Arquitectura documentada.

## Seguridad

- Permisos definidos.
- Secrets fuera del repo.
- Datos sensibles identificados.

## Drift

- Specs coinciden con codigo.
- Tasks coinciden con roadmap.
- Tests cubren reglas criticas.

## TODO por proyecto

- Definir frecuencia de revision.
- Definir indicadores.
