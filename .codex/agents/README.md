# Codex Agents

## Proposito

Mapear agentes documentados en `agents/` a agentes custom de Codex cuando el entorno lo soporte.

## Agentes sugeridos

- explorer
- documentator
- planner
- implementer
- tester-reviewer

## Reglas

- No duplicar reglas que contradigan `agents/`.
- Mantener nombres claros.
- Mantener alcance estrecho.
- Si el agente puede editar codigo, debe exigir spec y task.
